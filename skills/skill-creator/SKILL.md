# 技能创建技能

这个技能用于帮助另一个 Agent 创建、整理或升级技能。

## 目标

一个好技能应该做到三件事：

1. 触发条件清楚，让 Agent 知道什么时候该用它。
2. 内容足够具体，让 Agent 真正能照着做事。
3. 体积足够克制，不浪费上下文窗口。

## 设计原则

- 默认假设模型本身已经很强，只补充它原本不知道、但执行任务时真正需要的内容。
- SKILL.md 只保留核心工作流与判断依据，细节材料尽量拆到 `references/`。
- 高频、可重复、要求稳定的步骤，优先写成 `scripts/`。
- 会直接作为产出一部分使用的模板、图片、字体等，放进 `assets/`。

## 标准结构

```text
skill-name/
├── SKILL.md
├── scripts/
├── references/
└── assets/
```

## 创建流程

1. 先弄清楚技能要解决什么任务，以及用户会怎么触发它。
2. 抽取可复用内容，判断哪些应该写进 `SKILL.md`，哪些应该放到 `scripts/`、`references/`、`assets/`。
3. 编写 YAML frontmatter，至少包含 `name` 和 `description`。
4. 在正文里写清楚执行步骤、工具偏好、关键约束和异常处理。
5. 如果加了脚本，实际运行验证脚本是否可用。
6. 删除占位文件或无用说明，保持技能目录干净。

## 写作建议

- 用祈使句或直接操作指令，少写背景铺垫。
- 如果一个技能支持多种场景，把公共流程放在 SKILL.md，把分支细节放到不同引用文件。
- 不要额外创建 README、CHANGELOG 之类对 Agent 没帮助的文件。

## 触发描述

frontmatter 里的 `description` 很重要，因为它直接决定 Agent 是否会触发这个技能。描述里要同时写清：

- 这个技能能做什么
- 在什么任务或关键词下应该使用它

## 完成标准

当你结束技能编写时，请确认：

- frontmatter 正确
- 触发条件明确
- 步骤可执行
- 资源引用路径真实存在
- 没有多余文件
